## ⚠️ 運用上の注意事項

### 1. 未コミット変更の扱い
リポジトリには今も **500+ の未コミット変更**が蓄積（前セッション以前からのもの）。新しい PR 作成時は必ず **選別コミット**（該当ファイルのみ add）を徹底すること。

### 2. PSI 計測は chunked 必須
ops-notes §12 参照。`scripts/psi-one.mjs` を使い、bwrap サンドボックス制約下で 45s/call 以内に収める。同一 URL 間 60s+ 間隔。

### 3. tsc 単独ファイル実行の偽陽性
`tsc --noEmit <file>` で JSX 構造エラーが大量に出るが、tsconfig 経由の `npm run build` では通る既知現象。**最終検証は Claude Code の `npm run build` で実施**が正。

### 4. Vercel デプロイ反映は 2-3 分
マージ直後に本番確認すると 404 になることあり。80-180 秒待機してから curl で検証。

### 5. 外部リンク（セミナー告知など）は先方サイト
こちらから追加依頼はしない。主催者が自然に /kenji-eda URL を掲載してくれているので、その流れを壊さない。
